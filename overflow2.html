Jump Flood Outlines

    This is a post processing outline effect implemented in Unity's Universal Render Pipeline as a Scriptable Renderer Feature. It is based on this amazing article on the Jump Flood Algorithm by Ben Golus. The shader generates a realtime distance field to the boundaries of objects or textured lines and uses that to draw outlines of any width.

    In the past, I have made outline shaders that work by detecting edges in the depth buffer and surface normals, but I found them incredibly finicky and frustrating to tune properly. The goal with this project was to make an outline shader that is performant and allows for full artist control of outline placement and width. The first step is to render all objects in the "Outline" light layer to a silhouette buffer. They are rendered with a random per object value in the red buffer, and if present a special edge definition texture in the green and blue channels. The next pass prepares for the Jump Flood passes by detecting any place where the color in the red or green channels change, and returning only those pixels. The green channel texture is how I can define custom outlines. The Jump Flood algorithm is then run over several passes, which calculates the coordinate of the nearst seed pixel. In the final composite pass, the shader returns an outline pixel if the value in the distance field is less than the outline width. The outline width limit is multipled by the value in the blue channel, which allows for artist defined outline width, and the depth buffer, to make distant objects more legible.

Rowland's Adventure

This was my entry for a 2 day game jam about controlling gravity, mostly inspired by Super Mario Galaxy. The gravity effect was accomplished by detecting planets with large triggers, and then determining the direction of gravity with a specialized function that accounts for the player's relative position and the shape of the planet. I wanted to avoid the raycast method, as it has proven itself quite janky in the past. I modelled the character myself in Blender and animated it procedurally in C#. This way my first attempt at designing a level with a silent tutorial and a gradual difficulty progression. Playtests showed it wasn't quite effective, and unfortunately I ran out of time to fix it. I learned that if you wish to communicate to the player without words, you must use precise colors and symbols as your language instead.

Ray Tracing from Scratch

    While I had some experience, this was my first real project that taught me compute shaders. It renders a scene of randomly generated spheres of varying size, albedo, and specular. It does not use any Ray Tracing APIS or accelerators, everything is done purely through a compute shader.

    When the project is first run, it creates a buffer of 'Sphere' objects, and randomly assigns each object a radius, position, albedo, and specular value. Spheres that are intersecting are discared and re-rolled. This buffer is shipped off to the GPU with a render texture for output. Every frame, the compute shader is executed with one thread for every pixel on the screen. Each thread sends out a ray from the camera at its pixel's world location. The ray's intersection point with the ground plane and every sphere in the scene is calculated, and the closest one is chosen. They ray is reflected off the intersecting surface based on the surface normal. It continues this calculation on loop until it runs out of energy or hits the skybox. The light contribution for every bounce is then added up and written to the render texture. Finally, a simple blit shader reads from the render texture and outputs the value to the screen.

Unity 3D Graphing Library

    I could not find and 3D graphing tools for Unity, so I made my own. It uses compute shaders to calculate the position of every point based on the panning and scaling of the graph view, and GPU instancing to render each point in real time. The user can pan and zoom the chart with the mouse.

    At runtime, the library takes in an array of data points as floats for each data series the user wants graphed. These are packed into a compute buffer and sent to the GPU. A compute shader is dispatched with a thread for every point in the data set. Each thread creates a position for the data point at the correct position and rotation inside the graph view. Once this is done, I call DrawInstanced() on the CPU, and draw one sphere for every point. The point shader gets its position from the ComputeBuffer using its instance ID as the index, and offsets each sphere vertex in the vertex shader by that position. It also colors the point with a user-specified color sent from the CPU. The compute shader is only dispatched again if the user moves the chart, and the points are only drawn if the graph view is inside the view frustrum.

Serial Solitaire

This was a last minute entry into a game jam with a cozy autumn theme. Since I had so little time, I decided to make my own solitaire variant! Technically it is very simple, just screen raycasts and object parenting. My inspiration for the design came from my frustration of dealing the King cards in normal solitaire. They don't stack with anything, so they can only be moved if there is an empty slot. I removed the King and just required each stack to have 2 Queens, and Queens can stack on top of each other. I also tried something new by adding a stack to the side that can hold 5 cards, but they can only be retrieved in the order they were put in. I'm very happy with how the design plays, it feels difficult but not too difficult, and the stack forces some fun strategy! Everytime I open it up again I get sucked in for an hour. I made the card designs as well as the board background myself, just using tileable textures and an image editor, and I think it looks quite good! This game taught me that it is much easier to make an actually good game when you manage scope properly.

Soft Reminder

I needed a periodic, soft and silent reminder for therapy work, but I could not find any apps that allowed this, so I made one myself! This is a simple Android app that runs a custom vibration pattern at a user defined rate. An app must have a foreground process to use the vibration motor, so I had to start up a foreground service from the background repeatedly. I used Android's Alarm Scheduler API, since precise timing was important to me. This approach is surprisingly kind to battery life, probably because my foreground service is so simple. The UI was made in Jetpack Compose, which might be the best UI framework ever conceived, I love it.

Cookie Climber

This is the first finished game of the Video Game Design Club at my University! My role in the project was primarily to teach other members how to program in Unity, since I was one of the most experienced people in the club. I taught C#, the basics of Unity's framework, physics programming, state machines, and importing 2D assets. I did make some contributions fixing bugs, hooking up the UI, and making some of the sprites, but I was mostly focused on coaching. For being a lot of people's first ever game, I think it came out really well!

Desert Bus 3D

I used to have a friend a while ago who loved the Desert Bus Charity Marathon, so I made a 3d version of the game for him! It has a fully procedural terrain system based on heightmaps. It generates heightmap textures with a few octaves of Perlin Noise. The system also uses chunks with varying levels of detail to make far render distances possible! The terrain textures are just triplanar mapped in a shader. This project taught me so much, it is where my skills in many advanced game development topics began.

3D Game of Life

This was one of my very first Unity projects! It allows you to build and save any initial conditions for a simulation you like. You can also change the rules for the simulation along the top (Green for create, Yellow for Survive, Red for Death) for any number of neighbors. In addition to teaching me the basics of Unity, I also learned how to serialize data, write it to disk, and read it back later.

Wifi Website

My first university project was to make a website with a group, so we made one that monitors the wifi traffic of some popular campus buildings and estimates how good the signal will be. The goal was to help students find a good place to study. I made an Android App for it that used a REST API to pull data from the server, and then make helpful visualizations of that data for the user. This was my first serious Android development project, and I learned a lot about project structure and code organization as well as the Android framework itself.
