Rowland's Adventure 2

    Rowland's Adventure 2 is a 3D platforming game inspired by Super Mario Galaxy I made for my game engine architecture class. It was made in a barebones Java engine designed by the professor. I had to implement many features that I am used to having in Unity or Unreal into this engine myself. The game features a fully physics driven character I programmed from scratch. It suports different actions and states, such as jumping, falling, rolling, diving, and sliding. In addition, there are different surface materials in the game that all have different properties and effect how the character moves. There is a custom planetoid system that uses ghost colliders to detect which planet any physics object is closest to. It then calculates the correct force and direction of gravity based on the planet's properties. The game supports spherical and square shaped gravity fields of any strength. All the planets are driven by heightmaps done in the vertex shader. I wrote a custom lighting shader for all objects to get the cell shaded aesthetic of the game. The dialogue and NPC system was written from scratch to allow for dialogue pop ups and changing character animations. To get all this working, there is an entire document of fixes and additions I made primarily to the engine's rendering and physics systems.

Wooden Croaks Bookstore

This game was a group project that was a fantastical take on the game Unpacking, where you run a bookstore in a magic forest. I ended up doing a lot of work on the project. I programmed the holding and placing prop mechanics, which work by defining a grid in world space, and snapping objects to that grid based on how many squares their footprint occupies. I made a basic dialogue system that supported showing a full screen text prompt and triggering in game events. There was also a main sequencing system that worked on a day system, that allowed non-programmers to specify dialogue events and what props would be spawned for the player for each day in the game. I made an outline shader effect, which is a Scriptable Renderer Feature in Unity's URP that runs edge detection on all objects based on depth, normals, and color, and renders black pixels where the test passes.

Digital Instrument Cluster

    For my senior project, I was on a team developing a digital instrument cluster for an experimental desert race car for the school's engineering department. It is an embeded system project designed to run on both Windows and Linux written in Python and C++. The front end takes in data and uses it to draw a dashboard directly with OpenGL. I was responsible for all the backend data handling. I wrote a reader script that interfaces with our hardware device that pulls data from the car's data bus system, called CANBUS. This data goes through an interpretation script that I designed to allow future engineering students to add and program in additional sensors for the system. This data is also logged in real time to a .csv file. I made it format the data for easier data analysis, and the file is split up into multiple chunks if it gets too large to prevent crashes in Google Sheets. Since the car itself is still under construction, I also made a test reader and data that simulates real car data and pipes it into the dashboard.

GPU Instanced Grass

    I saw a GDC talk on making grass with instanced geometry, and wanted to give it a go myself. This effect uses GPU instanced grass blades positioned in a compute shader. It also calculates a wind texture in real time from stacked sine waves in another compute shader to animate every individual grass blade.

    When the project is first run, or whenever the grass object changes, the grass initialization compute shader is executed. It fills a Compute Buffer with Grass objects that contain information about the world position, wind uv coordinates, and size for each blade of grass. Every frame, a separate compute shader is executed to generate a wind noise texture. This is done using sine waves stacked at different frequencies to model large gusts of wind with smaller vortexes disturbing the grass. The resulting texture is sent to the grass blade material, and a call is made to render an instanced grass blade mesh for every entry in the grass blade buffer. The grass model itself is a long pointy triangle with 3 horizontal subdivisions. In the vertex shader, the grass blade is moved to the position stored in the grass blade buffer. It then offsets each vertex in the specified wind direction depending on the value of the wind texture at its position and how high the vertex is up off the ground. Each blade is also given a random width and height in a specified range so they are not all identical. The lighting model is very simple, all it does is interpolate between a tip color, blade color, and base color, and multiplies that by a simple Lambertian Diffuse factor.
